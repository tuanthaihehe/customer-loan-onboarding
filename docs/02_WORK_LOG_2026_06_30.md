### 12.1. Nhận xét tổng quan

Logic trong Excel phù hợp để implement tính năng đề xuất sản phẩm vay sau khi đã định giá tài sản.

Flow chính:

```text
1. Có adjustedAssetValue = AssetValuation.FinalValueAmount.
2. Lọc sản phẩm theo loan purpose.
3. Lọc theo asset type, tenor, score grade.
4. Tính max loan theo LTV.
5. Tính effective max loan.
6. Loại sản phẩm nếu effective max loan < min loan.
7. Tính suggested loan amount.
8. Tính loan amount gap.
9. Tính estimated monthly payment.
10. Sort và auto chọn sản phẩm tốt nhất.
```

### 12.2. Công thức chính

```text
maxLoanByLtv = adjustedAssetValue * product.maxLTV
```

```text
effectiveMaxLoanAmount = min(maxLoanByLtv, product.maxLoanAmount)
```

```text
Nếu effectiveMaxLoanAmount < product.minLoanAmount
=> loại sản phẩm
```

```text
suggestedLoanAmount = max(product.minLoanAmount, min(requestedLoanAmount, effectiveMaxLoanAmount))
```

```text
loanAmountGap = abs(suggestedLoanAmount - requestedLoanAmount)
```

```text
principalPerMonth = suggestedLoanAmount / selectedTenor
```

```text
interestPerMonth = suggestedLoanAmount * interestRatePerMonth
```

```text
estimatedMonthlyPayment = principalPerMonth + interestPerMonth
```

Sort:

```text
1. loanAmountGap tăng dần
2. estimatedMonthlyPayment tăng dần
```

### 13.6. Logic AssetService

File chính:

```text
AssetServiceImpl
```

Logic `lookup`:

- Nhận `assetType` và `licensePlate`.
- Validate asset type và biển số bắt buộc.
- Chuẩn hóa biển số về uppercase.

- Tìm asset theo biển số:
  - Nếu chưa có, tạo asset mới.
  - Nếu có, cập nhật variant nếu cần.
- Nếu asset không còn trạng thái `AVAILABLE` và hồ sơ chưa có asset, chặn bằng lỗi `ASSET_ALREADY_PLEDGED`.
- Lưu asset.
- Gắn asset vào `loan_application.asset_id`.
- Trả `AssetSnapshotResponse`.

### 13.7. Logic AssetValuationController/Service

File chính:

```text
AssetValuationController
AssetValuationService
AssetValuationServiceImpl
```

Đã tách rõ 3 API:

```http
GET /api/v1/asset-valuations/market-price?vehicleVariant=...
```

Logic:

- Nhận `vehicleVariant`.
- Resolve `VehicleVariant`.
- Tìm giá hiện hành trong `vehicle_market_price`.
- Trả giá xe gốc để UI hiển thị.

```http
POST /api/v1/asset-valuations/preview
```

Logic:

- Không cần `applicationCode`.
- Không lưu DB.
- Lấy `vehicleVariant` từ `assetSnapshot`.
- Tìm giá thị trường hiện hành.
- Lấy deduction type theo `deductionItems`.
- Tính tổng giảm trừ:

```text
totalDeductionAmount = sum(asset_deduction_type.deduction_amount)
```

- Nếu tổng giảm trừ lớn hơn giá thị trường thì báo lỗi.
- Tính giá trị tài sản sau định giá:

```text
finalValue = marketValue - totalDeductionAmount
```

- Trả response preview.

```http
POST /api/v1/loan-applications/{applicationCode}/asset-valuations
```

Logic:

- Tìm hồ sơ vay theo `applicationCode`.
- Kiểm tra hồ sơ đã gắn asset.
- Tính lại valuation bằng cùng logic với preview.
- Lưu `asset_valuation`.
- Lưu từng dòng `asset_valuation_deduction`.
- Trả kết quả đã lưu với trạng thái `VAL_ACTIVE`.

### 13.8. Logic deduction

Deduction hiện đang lấy từ bảng:

```text
asset_deduction_type
```

Frontend gửi:

```json
{
  "type": "OLD_VEHICLE",
  "rate": 0
}
```

Backend hiện dùng `type` để tìm `AssetDeductionType`.

Ở phiên bản hiện tại:

- `rate` chưa được dùng để tính tiền.
- Số tiền giảm trừ lấy từ `asset_deduction_type.deduction_amount`.

Lý do:

- DB design mới đang mô tả deduction theo khoản tiền cố định.
- Nếu sau này nghiệp vụ cần deduction theo tỷ lệ phần trăm, cần bổ sung schema như:
  - `deduction_method`
  - `deduction_rate`
  - `deduction_amount`
  - min/max cap nếu cần.

### 13.9. Logic LTV đã tách khỏi valuation

Trước đó code có tính:

```text
loanableValue = finalValue * 70%
```

Nhưng sau khi phân tích lại, logic này không thuộc asset valuation.

Đã chỉnh:

- `AssetValuationServiceImpl` không hardcode LTV nữa.
- `preview` và `save valuation` chỉ tính giá trị tài sản.
- LTV sẽ thuộc `LoanProduct`.
- Mức vay tối đa sẽ được tính ở bước đề xuất/chọn sản phẩm vay.

Logic sau này:

```text
maxLoanByProduct = assetFinalValue * loanProduct.maxLTV
```

### 13.12. Logic sản phẩm vay mới chỉ phân tích, chưa implement

Phần đề xuất sản phẩm vay từ Excel hiện mới dừng ở mức phân tích nghiệp vụ và đề xuất thiết kế.

## 15. Việc nên làm tiếp theo

Các bước tiếp theo hợp lý:

1. Thiết kế bảng `loan_product` và các bảng điều kiện áp dụng sản phẩm.
2. Viết migration Flyway cho loan product.
3. Seed sản phẩm từ file Excel sau khi làm sạch dữ liệu.
4. Thêm entity/repository/service/controller cho loan product recommendation.
5. Bổ sung API chọn sản phẩm vay cho loan application.
6. Tách logic scoring/score grade thành module riêng nếu cần dùng ở màn 5.
7. Bổ sung test cho:
   - lọc sản phẩm theo purpose/type/tenor/score.
   - tính LTV/effective max loan.
   - loại sản phẩm dưới min loan.
   - sort recommendation.
