## 8. Cập nhật `LoanApplicationServiceImpl`

### File đã sửa

```text
backend/loan-onboarding/src/main/java/com/f88/loanonboarding/service/impl/LoanApplicationServiceImpl.java
```

### Logic trước thay đổi

Khi save draft, service lưu trực tiếp:

```java
application.setLoanPurpose(request.loanRequest().loanPurpose());
application.setLoanTermMonths(request.loanRequest().requestedTenure());
```

### Logic sau thay đổi

Khi save draft:

1. Validate request bằng rule hiện có.
2. Tìm `LoanPurpose` theo `request.loanRequest().loanPurpose()`.
3. Tìm `LoanTerm` theo `request.loanRequest().requestedTenure()`.
4. Nếu không tìm thấy catalog tương ứng thì trả lỗi `RESOURCE_NOT_FOUND`.
5. Lưu:

```text
loan_purpose_id
loan_term_id
loan_term_months
requested_amount
```

### Lý do thay đổi

Schema mới không còn cột `loan_purpose` dạng text. Nếu không sửa service thì backend sẽ compile hoặc runtime lệch với database sau khi Flyway chạy V7.

### Logic trước thay đổi

```text

```

### Logic sau thay đổi

Rule chỉ cho phép các kỳ hạn đúng với catalog và constraint DB:

```text
3, 6, 9, 12, 18, 24
```

### Lý do

DB V7 có constraint:

```text
loan_term_months IN (3, 6, 9, 12, 18, 24)
```

Nếu rule Java vẫn cho kỳ hạn khác, request có thể qua service nhưng fail ở DB. Sửa rule giúp lỗi trả ra đúng tầng nghiệp vụ hơn.
