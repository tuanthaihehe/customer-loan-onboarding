### Logic trước thay đổi

Khi save draft, service lưu trực tiếp:

```java
application.setLoanPurpose(request.loanRequest().loanPurpose());
application.setLoanTermMonths(request.loanRequest().requestedTenure());
```

### Logic sau thay đổi

Khi save draft:

1. Validate request bằng rule hiện có.
2. Tìm `LoanPurpose` theo `request.loanRequest().loanPurpose()`.
3. Tìm `LoanTerm` theo `request.loanRequest().requestedTenure()`.
4. Nếu không tìm thấy catalog tương ứng thì trả lỗi `RESOURCE_NOT_FOUND`.
5. Lưu:

```text

loan_term_id
loan_term_months
requested_amount
```

Schema mới không còn cột `loan_purpose` dạng text. Nếu không sửa service thì backend sẽ compile hoặc runtime lệch với database sau khi Flyway chạy V7.

### Logic API recommendation

API:

```http
POST /api/v1/loan-products/recommendations
```

Logic xử lý:

1. Lấy danh sách loan product active.
2. Lọc theo `selectedLoanPurpose`.
3. Lọc theo `selectedAssetType`.
4. Lọc theo `selectedTenor`.
5. Nếu có `scoreGrade`, lọc thêm theo score.
6. Tính:

```text
maxLoanByLtv = adjustedAssetValue * maxLtvPercent / 100
```

7. Tính:

```text
effectiveMaxLoanAmount = min(maxLoanByLtv, productMaxLoanAmount)
```

8. Loại sản phẩm nếu:

```text
effectiveMaxLoanAmount < minLoanAmount
```

9. Tính:

```text
suggestedLoanAmount = max(minLoanAmount, min(requestedLoanAmount, effectiveMaxLoanAmount))
```

10. Tính:

```text
loanAmountGap = abs(suggestedLoanAmount - requestedLoanAmount)
```

11. Tính:

```text
principalPerMonth = suggestedLoanAmount / selectedTenor
interestPerMonth = suggestedLoanAmount * monthlyInterestRatePercent / 100
estimatedMonthlyPayment = principalPerMonth + interestPerMonth
```

12. Sort theo:

```text
loanAmountGap tăng dần
estimatedMonthlyPayment tăng dần
productCode tăng dần
```

13. Lấy top 3 sản phẩm.
14. Sản phẩm đầu tiên được đánh dấu:

```text
recommended = true
```
